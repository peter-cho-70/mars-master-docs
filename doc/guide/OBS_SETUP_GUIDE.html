WebSocket 서버 설정 (NBS 연동 필수)
이 설정을 하지 않으면 NBS 방송진행 앱에서 OBS를 제어할 수 없습니다.
1
메뉴 열기
OBS 상단 메뉴 → 도구 → WebSocket 서버 설정
2
서버 활성화
WebSocket 서버 활성화 체크박스 ON
서버 포트: 4455 (기본값 유지)
3
인증 설정
사내 네트워크: 인증 활성화 OFF (간편 연결)
외부 접속 허용 시: 인증 활성화 ON → 비밀번호 설정 후 NBS 앱에 동일하게 입력
4
시스템 콘솔 로그 활성화
디버그 로그 활성화 OFF (운영 중에는 꺼두기)
5
확인 및 저장
확인 버튼 클릭 → OBS 재시작 불필요 (즉시 적용)
6
연결 테스트
NBS 앱 열기 → 상단 "OBS 미연결" 버튼 클릭 → localhost:4455 입력 → 연결
상단 도트가 초록색으로 바뀌면 성공
3
스튜디오 모드 (PVW/PGM 분리)
스튜디오 모드는 프리뷰(PVW)에서 먼저 씬을 확인한 후 온에어(PGM)로 전환하는 방송 핵심 기능입니다.
1
스튜디오 모드 활성화
OBS 메뉴 → 보기 → 스튜디오 모드 클릭
또는 화면 오른쪽 하단 스튜디오 모드 버튼
2
화면 확인
좌측 = 프리뷰(PVW) — 다음에 나올 씬
우측 = 프로그램(PGM) — 현재 온에어 중인 씬
3
전환 버튼 설정
PVW와 PGM 사이 전환 버튼 클릭 시 두 화면이 교체됨
NBS 앱의 CUT/AUTO 버튼이 이 전환을 원격으로 실행합니다
스튜디오 모드가 꺼진 상태에서는 씬 클릭 즉시 온에어 됩니다. 반드시 스튜디오 모드 ON 상태로 방송하세요.

소스 생성 — 이름 정확히 입력 필수
소스 이름이 다르면 NBS에서 제어가 되지 않습니다. 아래 이름을 정확히 사용하세요.

5-1. 카메라 소스 (CAM1~4)

각 씬에 카메라를 추가할 때 공유 소스로 생성하면 여러 씬에서 동일 소스를 참조할 수 있습니다.

1
CAM1_ANCHOR 씬 선택
왼쪽 씬 목록에서 CAM1_ANCHOR 선택
2
소스 추가
소스 패널 하단 + → 비디오 캡처 장치
3
소스 이름 입력
"새로 만들기" 선택 후 이름 입력:
CAM1 (앵커1 메인 카메라)
CAM2 (앵커2 서브 카메라)
CAM3 (와이드 카메라)
CAM4 (날씨/클로즈업 카메라)
4
장치 선택
연결된 USB 캡처카드 또는 NDI 장치 선택 → 확인
5
다른 씬에 재사용
다른 씬에서 + → 비디오 캡처 장치 → "기존 항목 추가" → CAM1 선택
(복사가 아닌 동일 소스 공유 — 설정 변경이 모든 씬에 반영됨)

5-2. 서버(미디어) 소스 (SERVER1~3)

1
SRV1_VCR 씬에서 소스 추가
+ → 미디어 소스
2
소스 이름 및 설정
이름: SERVER1
로컬 파일: 임시 테스트 파일 선택 (나중에 NBS에서 자동 변경됨)
반복: OFF (단일 재생)
비활성화 시 숨기기: ON
3
SERVER2, SERVER3도 동일하게 생성
SRV2_VCR, SRV3_VCR 씬에서 각각 반복

5-3. 오디오 입력 소스 (앵커 MIC)

소스 이름 (정확히 입력)소스 타입연결 장치추가할 씬
앵커1_마이크오디오 입력 캡처오디오 인터페이스 채널 1모든 CAM* 씬

오디오 소스는 씬 내부에 추가하지 않고 오디오 믹서에서 전역으로 관리해도 됩니다. 단, 소스명은 반드시 위와 동일해야 NBS의 자동 볼륨 제어가 작동합니다.
