### "설명서 없이 바로 쓰는" 라이브 방송 큐시트 에디터 (vMix 4K 연동)

| 항목 | 내용 |
|---|---|
| 문서명 | Cuesheet-UI.md |
| 상위 문서 | Live.md (LPP 전체 기획서) |
| 범위 | 큐시트(러다운) 제작 화면의 UI/UX 설계 |

## 1. 설계 철학 — "이미 아는 것처럼 느끼게 하라"

### 1.1 3대 원칙

**① 친숙성 우선 (Jakob's Law)**
사용자는 다른 사이트/프로그램에서 쌓은 경험을 그대로 기대한다. 큐시트 UI는 **새로운 규칙을 가르치지 않고, 이미 널리 쓰이는 소프트웨어의 조작법을 그대로 가져온다.** 엑셀처럼 셀을 편집하고, 트렐로처럼 카드를 끌고, 파워포인트처럼 슬라이드를 정렬한다.

**② 점진적 노출 (Progressive Disclosure)**
처음 화면은 단순하게. 고급 옵션(오디오 버스, 믹스마이너스, PIP 레이아웃)은 필요할 때만 펼친다. 초보자는 "제목 + 유형"만으로 큐를 만들고, 숙련자는 속성 패널로 깊게 들어간다.

**③ 즉시성 (No Modal, No Wait)**
큐 추가·편집은 팝업창을 거치지 않고 **그 자리에서** 일어난다. 클릭 한 번 줄이는 것이 매뉴얼 한 페이지를 없앤다.

### 1.2 "설명서 없는 UI" 체크리스트
- [ ] 모든 아이콘에 **텍스트 레이블 병기** (모호한 아이콘 단독 사용 금지)
- [ ] 유형마다 **일관된 색상** (한 번 배우면 어디서나 통함)
- [ ] 빈 화면에 **예시 + "첫 큐 추가" 버튼** (empty state가 곧 튜토리얼)
- [ ] 입력란마다 **placeholder 예시 문구**
- [ ] 파괴적 동작 뒤 **"실행취소" 스낵바** (실수해도 안전)
- [ ] 드래그 시 **드롭 위치 표시선** (어디에 놓일지 보임)
- [ ] 누락·오류는 **빨간 배지로 그 자리에서** 표시

## 2. 차용 소프트웨어 매핑 (핵심)

각 기능을 어느 소프트웨어의 어떤 패턴에서 가져왔는지 명시한다. **개발자는 "새로 발명"하지 않고 해당 패턴을 재현**하면 된다.

| 기능 | 차용 SW | 재현할 패턴 |
|---|---|---|
| 큐 목록(표) 편집 | **Excel · Google Sheets** | 셀 인라인 편집, Tab/화살표 이동, 복사·붙여넣기, 아래로 채우기 |
| 큐 순서 변경 | **Trello · Notion** | 행/카드 드래그앤드롭, 드롭 위치 미리보기 선 |
| 카드형 보기 | **PowerPoint 슬라이드 정렬** | 썸네일 그리드, 드래그로 재배열 |
| 새 큐 빠른 추가 | **Todoist · Slack** | 하단 "+ 큐 추가" 빈 줄, Enter로 연속 입력 |
| 큐 유형 선택 | **Notion 슬래시 명령** | `/` 입력 → 카메라·자막·영상·인터뷰 목록 |
| 전역 검색·실행 | **VS Code · Linear** | `Cmd/Ctrl + K` 명령 팔레트 |
| 소스 배치 | **OBS · Figma** | 좌측 소스 팔레트에서 드래그해 큐에 투입 |
| 속성 편집 | **Figma · PPT 서식창** | 우측 컨텍스트 패널 (선택 항목에 반응) |
| 진행/미리보기 | **vMix · OBS** | Preview→Program, 스페이스바=다음 큐 |
| 시간 합산 | **Excel** | 하단 합계 행, 자동 러닝타임 |
| 유형 색상 태그 | **캘린더 · 칸반** | 색상칩으로 한눈에 구분 |
| 되돌리기/복제 | **공통 표준** | `Cmd+Z` / `Cmd+D` |
| 템플릿 시작 | **PPT 새 슬라이드 · Notion 템플릿** | 갤러리에서 표준 진행 문법 삽입 |

## 3. 화면 레이아웃

**3-패널 + 하단 진행 바** 구조. OBS·Figma·프리미어가 공통으로 쓰는, 사용자가 이미 익숙한 배치다.

```
┌──────────────────────────────────────────────────────────────────────┐
│  ⌘K 검색   [편집 모드 | 리허설 | 라이브]        회차: 6/12 · 총 42:30  │  ← 상단 바
├────────────┬────────────────────────────────────────────┬────────────┤
│ 소스 팔레트 │            큐 리스트 (메인)                  │  속성 패널  │
│ (OBS 스타일)│         [표 보기 ▸] [카드 보기]              │(Figma 스타일)│
│            │ ┌──┬────┬─────────────┬──────┬─────┬──────┐  │            │
│ 📷 카메라1  │ │ # │유형│    제목      │ 소스 │시간 │오디오│  │ ▸ 선택된 큐 │
│ 📷 카메라2  │ ├──┼────┼─────────────┼──────┼─────┼──────┤  │  이름자막   │
│ 📷 카메라3  │ │ 1 │🎬 │ 오프닝 타이틀 │opening│0:15 │ 🎵M  │  │            │

│ 💬 상단자막 │ │ 4 │🎞 │ VOD 리포트   │Server│2:00 │ 🎵M  │  │ 직함:      │
│ 💬 이름자막 │ │ 5 │📷 │ 스튜디오 복귀│ Cam2 │1:00 │ 🎤1-3│  │ [경제부…  ]│

│  ◀ 이전   ▶ 다음 큐(Space)   현재:③ 이름자막   [PIP▾] [자막ON] [패닉]  │  ← 진행 바
```

- **좌측 소스 팔레트** — OBS의 소스 목록처럼 항상 보이며, 드래그해서 큐 리스트에 떨어뜨리면 카드가 생긴다.
- **중앙 큐 리스트** — 기본은 엑셀형 **표**, 상단 토글로 파워포인트형 **카드 그리드**로 전환.
- **우측 속성 패널** — Figma·파워포인트 서식창처럼, 선택한 큐에만 반응해 세부 설정을 보여준다.
- **하단 진행 바** — 편집이 끝나면 그대로 진행 콘솔이 된다(3장 상단 모드 토글과 연동).

## 4. 핵심 인터랙션 상세

### 4.1 큐 추가 — 4가지 익숙한 방법
사용자가 어떤 프로그램에 익숙하든 하나는 통하도록 **복수 진입점**을 제공한다.

1. **하단 빈 줄 클릭** → 바로 입력 (Todoist/Slack 방식)
2. **`/` 입력** → 유형 드롭다운 (`/카메라`, `/자막`, `/인터뷰`) (Notion 방식)
3. **소스 팔레트에서 드래그** → 원하는 위치에 드롭 (OBS/Figma 방식)
4. **⊞ 템플릿** → 표준 진행 문법 한 벌 삽입 (PPT 새 슬라이드 방식)

### 4.2 큐 편집 — 엑셀처럼
- 셀 **더블클릭 또는 Enter**로 편집 시작, **Tab**으로 다음 칸, **↑↓**로 행 이동.
- **복사(Ctrl+C)·붙여넣기(Ctrl+V)**, **아래로 채우기(Ctrl+D)** 로 반복 큐 대량 생성.
- 자막 문안·직함처럼 자주 바뀌는 값은 표에서 바로 인라인 편집.

### 4.3 순서 변경 — 트렐로처럼
- 행 왼쪽 **⣿ 핸들**을 잡고 드래그, 놓일 위치에 **파란 선** 표시.
- 카드 보기에서는 파워포인트 슬라이드처럼 그리드에서 재배열.

### 4.4 뷰 전환 — 데이터를 3가지 시선으로
| 뷰 | 차용 | 언제 |
|---|---|---|
| **표 보기** | 엑셀 | 정밀 편집, 시간·오디오 일괄 관리 |
| **카드 보기** | PPT 슬라이드 정렬 | 전체 흐름·구성 조망 |
| **타임라인 보기** | 다빈치 리졸브 | 시간 축으로 러닝타임 균형 확인 |

> 동일 데이터, 뷰만 전환. 다빈치 리졸브를 다뤄본 사용자에겐 타임라인이, 문서 작업자에겐 표가 자연스럽다.

### 4.5 명령 팔레트 (`Cmd/Ctrl + K`)
VS Code·Linear처럼 어디서든 호출. "이름자막 추가", "6번 큐로 이동", "소재 생성" 등을 타이핑으로 실행 → **마우스 없이도 전체 조작 가능**.

### 4.6 소재 자동생성 연결
표를 다 짜면 상단 **[소재 생성]** 한 번으로 `show.vmix`·타이틀·데이터소스·큐명령이 생성된다(Live.md 7장). UI는 **"큐시트를 짜는 것"에만 집중**하고, vMix 변환은 뒤에서 처리된다.

## 5. 큐 카드 체계 (유형 · 색상 · 아이콘)

한 번만 익히면 어디서나 통하는 **일관된 시각 언어**.

| 유형 | 아이콘 | 색상 | 주요 속성 |
|---|---|---|---|
| 타이틀 | 🎬 | 진회색 | 오프닝/끝, 전환(Fade) |
| 카메라 | 📷 | 파랑 | 카메라 번호(1~4), 마이크 |
| 상단자막 | 💬 | 노랑 | 문안, Overlay 1 |
| 이름자막 | 💬 | 주황 | 이름·직함·소속, Overlay 2 |
| 서버영상 | 🎞 | 보라 | 파일, 오디오 M 인가 |
| PIP | 🖼 | 청록 | 소스(PPT/PC/영상/라이브), 배치 |
| 기타영상 | 🎞 | 연보라 | 파일/이미지/웹 |
| 전화 인터뷰 | ☎ | 초록 | 오디오만, 믹스마이너스 버스 A |
| 화상 인터뷰 | 📹 | 초록 | vMix Call, 리턴 출력·버스 |

- 색상칩은 표의 유형 열과 카드 테두리에 동일 적용 → **스캔 한 번에 구성 파악**.

## 7. 편집 → 라이브 모드 전환

상단 토글 하나로 **같은 큐시트가 진행 콘솔로 변신**. 새 화면을 배울 필요가 없다.

- **편집 모드**: 표/카드 자유 수정.
- **리허설 모드**: 큐를 눌러보며 vMix Preview로 점검(방송 송출 없음).
- **라이브 모드**: **스페이스바 = 다음 큐**(vMix/OBS와 동일 감각). 현재 큐 하이라이트, TALLY 실시간 표시, PIP 소스 전환·자막 ON/OFF·**패닉 컷**(즉시 카메라 복귀) 버튼.

| 패닉 컷 | `Esc Esc` |

## 9. 접근성 · 반응형

- **아이콘+레이블 병기**, 색상만으로 정보 전달하지 않음(색약 대응: 아이콘·텍스트 보강).
- 키보드만으로 전체 흐름 완주 가능(마우스 비의존).
- 데스크톱 3-패널 → 노트북/좁은 화면에서는 좌·우 패널 접힘(Figma 반응형 패턴).
- 부조종실 이중 모니터 대응: 진행 콘솔을 별도 창으로 분리 가능(vMix 데이터소스 창 방식).

## 10. MVP 우선순위 (무엇부터 만들까)

| 순위 | 기능 | 이유 |
|---|---|---|
| 1 | 표 기반 큐 리스트 + 인라인 편집 + 드래그 정렬 | 큐시트의 본질, 엑셀/트렐로 감각 |
| 2 | 소스 팔레트 드래그 투입 + 유형 색상 체계 | "설명서 없이" 이해되는 핵심 |
| 3 | 표준 진행 문법 템플릿 삽입 | 빈 화면 공포 제거, 즉시 시작 |
| 4 | 속성 패널(자막·PIP·오디오) | 점진적 노출로 심화 |
| 5 | 편집→라이브 모드 + 스페이스 진행 | vMix 연동 진행의 완성 |
| 6 | 명령 팔레트 · 타임라인 뷰 | 숙련자 가속(후순위) |

---

*큐시트 UI는 "새 도구를 배운다"가 아니라 "아는 도구를 조합해 쓴다"는 감각을 목표로 한다. 화면의 모든 조작은 엑셀·노션·트렐로·파워포인트·OBS·다빈치 중 하나에서 이미 경험한 방식이어야 한다.*
