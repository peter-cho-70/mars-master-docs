# Master — 뉴스 방송 시스템

MARS(큐시트 제작)와 Master(장비 제어) 두 앱으로 구성된 뉴스 방송 자동화 시스템입니다.

---

## 구성 파일

| 파일 | 역할 |
|------|------|
| `nps-news-system.html` | MARS — 기사 입력 및 큐시트 제작 |
| `news-broadcast-system.html` | Master — vMix 장비 제어 |
| `obs/news-broadcast-system-obs.html` | Master — OBS 병렬 버전 (OBS 전용 파일은 `obs/`에 모아둠) |
| `cg-operator.html` | 자막 담당 — Master를 거쳐 vMix로 자막 요청을 중계하는 독립 화면 |
| `index.html` | 빠른 시작 메뉴 |
| `server.py` | MARS↔Master 큐시트 공유 서버 · Videohub 릴레이 · CG 요청 큐 |

## 실행 방법

### 단일 PC (혼자 다 쓸 때)

서버 없이 브라우저에서 직접 파일을 엽니다.

```bash
open index.html
```

`index.html`에서 원하는 앱을 선택합니다.

- **방송 제어 (Master)** → `news-broadcast-system.html`
- **큐시트 제작 (MARS)** → `nps-news-system.html`

---

### 네트워크 모드 (MARS·Master 다른 PC)

MARS(기자 PC)와 Master(제어 PC)가 별도 컴퓨터일 때 사용합니다.

**1. Master PC에서 서버 실행**

```bash
cd /path/to/autoplay
python3 server.py
```

> 기본 포트는 **8080**입니다. 변경하려면 `python3 server.py 9000` 처럼 포트를 인자로 전달합니다.

**2. Master PC의 IP 확인**

```bash
ipconfig getifaddr en0
```

**3. 각 PC에서 브라우저로 접속**

| PC | 접속 주소 |
|----|----------|
| MARS (기자 PC) | `http://<Master IP>:8080/nps-news-system.html` |
| Master (제어 PC) | `http://<Master IP>:8080/news-broadcast-system.html` |

## 처음 사용 시 (vMix 연동)

Master는 vMix REST API로 카메라·영상 서버·CG를 제어합니다. (OBS 연동은 `obs/news-broadcast-system-obs.html`(병렬 버전)에서 별도로 관리 — OBS 전용 파일은 전부 `obs/` 폴더에 모여 있습니다.)

1. vMix를 켜고 **Web Controller**가 켜져 있는지 확인합니다 (기본 포트 8088).
2. Master의 **설정** 탭 → **vMix 연결**에서 vMix PC의 IP·포트를 입력하고 연결합니다.
3. **vMix 자동 설정**으로 카메라/서버 인풋을 스캔하거나, 인풋 매핑 표에서 직접 지정합니다.

## 단축키

뉴스 모드와 라디오 모드는 서로 완전히 독립된 단축키를 씁니다 — 같은 키라도 모드에 따라 다른 동작을 하며 서로 영향을 주지 않습니다. 단축키 목록에서 키를 클릭해 실제로 키보드를 눌러 지정하고(vMix 방식), 지정할 기능을 고릅니다.

- **뉴스 모드**: 큐시트 옆 단축키 패널 (자막·서버 소재 재생 등)
- **라디오 모드**: 기존 단축키 탭

## 화면 전환 효과 (전환 In / 전환 Out)

아이템 사이 전환 효과는 타입별 기본 원칙을 MARS·Master가 공유하고, 필요하면 아이템별로 예외를 줄 수 있습니다.

| 타입 | 전환 In (진입) | 전환 Out |
|------|----------------|----------|
| 리포트 | Cut | Cut — 앵커멘트 후 영상으로 전환 |
| 단신 | Mix(디졸브) | Mix(디졸브) — 앵커멘트 후 영상으로 전환 |
| CM | Mix(디졸브) | Mix(디졸브) — 참고용 (아래 설명 참고) |
| 브리지 | Cut | Mix(디졸브) — 참고용 (아래 설명 참고) |

- **리포트·단신(순차형)**: 전환 In = 앵커 카메라로 들어올 때, 전환 Out = 앵커멘트 후 "영상전환" 버튼으로 영상으로 넘어갈 때 적용됩니다.
- **CM·브리지 등(비순차형)**: 실제로 화면에 적용되는 건 **각 아이템 자신의 전환 In**뿐입니다 — 아이템 진입 효과는 항상 그 아이템 자신의 전환 In을 따르고, 직전 아이템이 무엇이든(전환 Out을 뭘로 설정했든) 영향을 주지 않습니다. 비순차형 타입의 전환 Out 값은 "이 타입에서 나갈 때는 보통 이런 효과가 어울린다"는 참고용 기본값일 뿐, 자동으로 다음 아이템에 적용되지 않습니다. 특정 전환에서 디졸브로 나가고 싶으면 **다음에 오는 아이템 쪽의 전환 In**을 디졸브로 설정하세요.
- 타입별 기본값은 "타입 관리" 모달(PDF/설정 탭)에서, 개별 아이템 예외는 아이템 편집창의 "전환 In"/"전환 Out"에서 설정합니다. 편집창에는 지금 값이 타입 기본값과 다른지 힌트로 보여줍니다.
- 전환 효과 자체(지속시간, Wipe 외 커스텀 효과 등)는 Master의 **설정 → 전환효과 설정**에서 자유롭게 추가·수정합니다.

## 실사용하지 않아 Master UI에서 제거한 기능

아래 기능들은 실제 방송 운용에서 쓰이지 않아 Master(`news-broadcast-system.html`)의 탭 구성에서 제거했습니다. 코드 자체는 삭제됐으므로 필요해지면 git 히스토리에서 복원해야 합니다.

- **탈리(Tally) 탭** — PGM/PVW 상태 큰 표시, 소스별 탈리 그리드, 오디오 레벨 미터(실제 오디오가 아닌 시뮬레이션 표시였음). 실제 PGM/PVW 상태는 스위처 탭에서 계속 확인 가능합니다.
- **미디어 소스 제어** (미디어 탭) — 소스명/클립 경로를 직접 입력해 재생/일시정지/정지/처음부터/클립 로드/상태 확인을 수동으로 호출하던 패널. vMix 전환 이후에는 SRV1-3 리스트 기반 재생(`vMix 리스트 소재 재생 테스트`)으로 대체되어 이 패널은 쓰이지 않았습니다.
- **클립-큐시트 자동 매칭** (미디어 탭) — `아이템번호_기자이름.mp4` 명명 규칙으로 빈 클립을 자동 채워주던 기능.
- **기사 매칭** (미디어 탭, Master 내 수동 입력) — MARS에서 이미 작성한 기사를 Master에서 제목/담당자/본문으로 다시 입력하던 중복 기능. MARS 쪽 기사 작성·매칭이 정본이라 Master 쪽 수동 입력은 쓰이지 않았습니다. (단, MARS에서 넘어온 기사 데이터를 큐시트 목록의 📰 배지와 프롬프터 앵커멘트 표시에 쓰는 부분은 그대로 남아있습니다.)

## 변경 이력 (요약)

### 2026-07-15

- **전환효과 필수 프리셋 강제 + 자기 치유** — Cut/Mix(디졸브)/Fade/Wipe는 MARS·실제 vMix와 연동되는 필수 기본값이라 이제 삭제 자체가 막힙니다(예전엔 확인창만 띄우고 실제로는 삭제돼, 신규 아이템 전환In/Out에 Cut이 안 보이는 상태가 될 수 있었습니다). 저장된 프리셋 목록에서 이 4개 중 하나라도 빠져 있으면 다음 로드 때 자동으로 복구됩니다.
- **큐시트 아이템 더블클릭 편집** — 뉴스 큐시트 표에서 아이템 행을 더블클릭하면 기존 ✏️ 버튼과 동일하게 편집창이 바로 열립니다.
- **아이템 편집창 라우터 TAKE 버튼** — Videohub 출력/입력 셀렉트는 값만 저장해두고 실제 전환은 방송 중 온에어 시점에만 발사되던 방식이라, 편집 중 선택해도 아무 반응이 없었습니다. TAKE 버튼을 누르면 저장·방송과 무관하게 즉시 실제 크로스포인트가 전환됩니다.
- **자막 섹션 재배치** — 아이템 편집창에서 "자막" 섹션이 "기타" 섹션 바로 앞으로 이동했습니다.
- **중계/출연 시퀀스 단계 개별 편집** — 기존엔 "질문 수"만으로 앵커갈라치기→중계풀샷→서버갈라치기 패턴이 고정 반복됐는데, 이제 실제 실행되는 단계 목록(`liveSeq.phases`)을 아이템 편집창에서 직접 추가·삭제·순서변경·소스변경할 수 있습니다. "↻ 질문 수로 단계 자동 생성" 버튼으로 기존 방식대로 초안을 만든 뒤 특정 질문만 단계를 더하거나 빼는 식의 미세 조정이 가능합니다. 질문 수만 있는 기존 아이템/템플릿은 그대로 예전 방식대로 동작합니다(하위호환).
- **중계 풀샷(기자 원샷) RS1~6 스위처 소스 추가** — 라우터 크로스포인트 전환 자체는 이미 연동돼 있었지만 그 신호를 받는 vMix 인풋이 스위처 소스로 등록된 적이 없어 "기자 원샷"을 고를 방법이 없었습니다. RS1~RS6을 스위처 소스로 추가해 중계 풀샷 소스/기타 소스/시퀀스 단계 드롭다운 어디서나 선택할 수 있습니다(vMix 인풋명은 실제 캡처 인풋 구성 후 설정 탭에서 채워야 함).

### 2026-07-13

- **출연(스튜디오 게스트) 전용 LIVE 템플릿** — MARS `출`(출연)과 `중`(중계) 코드가 Master에서는 둘 다 LIVE 타입으로 합쳐져 있었는데, 실행 시퀀스 엔진(`liveSeq`)에 구분값(`kind`)을 추가해 서로 다른 진행 형태를 지원합니다. 중계는 기존대로 앵커→[앵커갈라치기→중계풀샷→서버갈라치기]×질문수→중계마무리, 출연은 앵커→[앵커갈라치기(DVE2)→기자→기자서버갈라치기(DVE1)]×질문수→Full Shot으로 마무리 — 아이템 편집창의 라벨도 자동으로 바뀝니다.
- **MARS 시퀀스 소스 사전 설정** — 중계/출연 아이템 편집창에 CAM1~4/SRV1~3/DVE1~3 중에서 각 단계 소스를 미리 지정하는 UI를 추가했습니다. MARS는 실제 vMix 인풋명을 모르므로 Master와 공유하는 표준 id로만 저장하고, Master로 넘어가면 초기값으로만 쓰입니다 — 방송 중 Master에서 한 번이라도 값을 바꾸면 그 뒤로는 MARS가 재전송해도 Master 값이 유지됩니다(라우터 사전 설정과 같은 원칙).
- **큐시트 "진행" 표시** — Master 큐시트 표의 각 아이템 아래에 실제 온에어 시 어떤 카메라/DVE/서버로, 어떤 전환효과(컷/디졸브 등)로 넘어가는지 한 줄씩 보여줍니다. 질문이 여러 개인 중계/출연 아이템은 "질문 1개 = 1줄"로 묶어 가로로 길어지지 않게 했습니다. 현재 진행 중인 아이템 기준 앞뒤 4개만 기본으로 펼쳐지고 나머지는 한 줄 요약으로 접혀(공간 절약), 클릭하면 개별적으로 펼치고 접을 수 있습니다.
- **아이템 편집창 "진행 미리보기"** — 편집창 맨 아래에 저장하기 전 지금 설정대로 실제 컷 순서를 보여주는 섹션을 추가했습니다. 라우터(Videohub RS# 크로스포인트)까지 포함해서 필드를 하나 바꿀 때마다(카메라/기타소스/클립/전환효과/시퀀스 소스/라우터) 실시간으로 갱신됩니다.
- **MARS 버그 수정** — 중계/출연 아이템 편집창에서 영상 클립을 지워 질문 수보다 적게 만들어도, 저장할 때마다 질문 수만큼 빈 슬롯이 도로 채워져 삭제가 무의미해지던 문제를 고쳤습니다. 질문 수는 최초 슬롯 생성에만 쓰이고, 그 뒤 클립 추가/삭제는 질문 수와 독립적으로 유지됩니다.

### Master

- vMix REST API 함수명 오류 수정 (`PlayInput`/`PauseInput`/`RestartInput`은 실제 API에 없는 이름 — `Play`/`Pause`/`Restart`가 맞음). 미디어 재생 제어가 전부 조용히 실패하고 있었습니다.
- SRV1-3 리스트 재생을 vMix에 이미 올라간 리스트 내용과 파일명으로 매칭하도록 변경 (리스트 자체를 새로 쓰지 않음). 클립을 재생할 때마다 `Function=Restart`를 먼저 보내 항상 처음(0초)부터 재생하도록 수정 — 같은 버튼을 재생 완료 후 다시 누르면 엉뚱한 영상이 나오던 문제 해결.
- vMix 리스트의 "Auto Next"가 클립이 끝나자마자 알파벳순 다음 파일로 자동 전환해버리는 문제(클로징에 엉뚱한 영상이 재생) → 클립의 실제 길이에 맞춰 자동 정지를 예약하도록 수정. "자동으로 다음 아이템 진행" 여부는 선택적으로 켜고 끌 수 있습니다.
- 단축키를 뉴스/라디오 모드별로 완전히 독립시키고, 키 캡처 방식(vMix 스타일)의 설정 UI를 추가.
- 탈리 탭·미디어 소스 제어·클립 자동매칭·기사 매칭 UI 등 실사용하지 않는 기능 제거 (위 "실사용하지 않아 제거한 기능" 참고).
- MARS→Master 큐시트 갱신 시, 방송이 실제로 시작되기 전이면 "방송적용" 확인 없이 바로 반영되도록 수정 (선택 여부가 아니라 실제 방송 시작 시각을 기준으로 판단).
- 아이템 전환효과: 커스텀 전환 프리셋(Wipe 등)이 적용되지 않던 문제 수정. 순차형(단신/리포트)의 "전환 In"(앵커 진입)과 "전환 Out"(앵커→영상)을 분리하고, 타입별 원칙(위 "화면 전환 효과" 참고)을 적용 — 아이템으로 들어올 때 효과는 항상 그 아이템 자신의 전환 In이 적용되도록 통일(직전 아이템에 따라 조용히 바뀌지 않음). 아이템 편집창에서 지금 설정이 원칙과 맞는지 힌트로 바로 확인 가능.
- 클립이 없는, 카메라도 없는 타입(CM/브리지 등)이 클립 누락 시 실수로 앵커 카메라(CAM2)로 컷되어 "앵커로 갔다가 간다"처럼 보이던 문제 → 클립이 없으면 화면을 바꾸지 않고 에러만 표시하도록 수정.

### MARS

- 아이템 편집창에 수동 "제목" 입력 필드 추가 (기사 매칭 자동화 이전까지 임시로 직접 입력/수정).
- 영상조회 탭이 가짜 샘플 데이터 대신 `media/` 폴더의 실제 파일을 보여주도록 변경.
- 미리보기(영상조회 탭, 아이템 편집창 클립 미리보기 모두) — 팝업 대신 화면 안에서 바로, 클릭 즉시 자동재생되도록 변경.
- 아이템 편집창의 클립 미리보기가 아이템을 저장 후 다시 열면 재생되지 않던 문제(임시 blob 링크가 세션마다 소멸) → `media/` 폴더의 실제 파일 URL로 대체 재생하도록 수정, 언제 다시 열어도 미리보기 가능.
- 전환 In/Out 필드와 원칙 확인용 힌트를 아이템 편집창에 추가 — Master와 같은 원칙을 공유.
- **자막(CG) 인/아웃 설정** — 자막마다 화면에 나타나고 사라질 시각(초)을 직접 지정할 수 있습니다. 실제 영상을 재생하며 정확한 지점에서 IN/OUT을 찍거나("▶ 자막 인/아웃 설정" 새 창), 타임라인 막대를 클릭·드래그해 조정합니다. 아이템 편집창에도 같은 기능이 2단(목록+영상) 레이아웃으로 내장되어 있으며, "저장"을 눌러야 반영되고 취소하면 다른 수정 내용과 함께 사라집니다. 단신은 영상 길이 기준 자동 계산(기본값 설정 가능), 리포트는 자막 추가 시 직전 자막 뒤로 자동 이어붙는 기본 타이밍이 채워집니다.
- 자막조회 탭의 "오버레이별 자막 미리보기" 패널을 접었다 펼 수 있게 변경, 자막/앵커멘트 목록이 길어지면 자체적으로 스크롤되도록 수정(기존 CSS 선택자 오타로 스크롤이 동작하지 않던 버그).

### vMix 연동(실제 장비 테스트로 발견/수정)

- 서버(SRV) 리스트 재생 시, 회전 대상이 아닌 타입(CM 등)이 항상 SRV1에 고정 배정되어 직전 아이템과 서버가 겹치는 문제 → 디졸브/와이프를 걸어도 같은 화면이라 컷처럼 보였습니다. 직전에 실제로 재생한 서버를 피해서 배정하도록 수정.
- 이 vMix 버전에 없는 `Function=Stop` 호출로 클립 자동정지 안전장치 2곳이 조용히 무력화되던 문제 → `Function=Pause`로 교체.
- 실제 방송처럼 클립을 전환 1초 전에 미리 재생 시작(프리롤)하고, 클립이 끝나기 2초 전에 자동으로 다음 아이템으로 전환하도록 자동화(초 단위 설정 가능, 기존 "자동 진행" 옵션 재사용).
